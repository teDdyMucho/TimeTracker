## 2. Apple rejection — handled correctly ⏳
- Apple rejected under **Guideline 3.2 (Business)**: the app is for a specific business's employees, but was set for **public** App Store distribution.
- **This is expected for a B2B app** — nothing wrong with the app itself, just the distribution type.
- **Replied to App Review** answering their 5 questions honestly and **requesting Unlisted App Distribution** (app stays on the App Store but is not publicly searchable — installs via a direct link shared with the client's crew). This is the right fit.
- **Now waiting** for Apple's response (~1–2 days) to enable unlisted distribution, then we resubmit.

## 3. New feature — Admin ↔ Employee Messaging ✅ (built)
- New **`messages` table** (migration `0014`) — 1-on-1 threads, RLS so employees see only their own thread.
- **Admin web:** new **Messages** tab — employee list with unread badges + a chat thread view (bubbles + composer). Sending pushes a phone notification to the employee.
- **Mobile:** a **Messages** screen (chat bubbles, composer, optimistic send, 8s polling) + a chat icon with unread badge on the home header.
- Fixed a Next.js render error (`revalidatePath` during render).
- **Keyboard fix:** on mobile the composer now lifts above the keyboard so you can see what you're typing (JS keyboard-height tracking, works in Expo Go; also set Android `softwareKeyboardLayoutMode: resize`).
