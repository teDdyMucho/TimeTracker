Big day on two fronts: (1) **built and shipped the installable Android app (APK)** so the client can try BuildOne on a real phone, and (2) **reworked Attendance into the single source of truth** — merged Timesheets into it, added Approve/Reject verification (with photo + GPS proof), and added filters across Attendance and Overtime. ## 1. Mobile App — Installable APK ✅ (built & shared)
- Prepared the build config (`eas.json` preview = APK, app name/icon/splash) and the client successfully **built the APK on EAS (Expo cloud)** under their own account.
- Documented the correct way to share it: use the direct **`.apk` artifact link** (installs on any Android phone, no login) rather than the private dashboard URL.
- Clarified that on Android the build is a **universal APK** — no per-device registration needed (that's iOS-only).

## 2. Mobile App — Polish ✅
- **Open animation**: on app launch, the BuildOne logo fades/scales in over a dark plate with a bronze underline sweep, then dissolves into the login page (which now also rises/fades in). One smooth "logo → login" motion.
- **Settings → Change password**: added **show/hide eye icons** on both password fields, and a **branded success modal** ("Password updated") replacing the plain OS alert.

## 3. Attendance = One Unified Tab ✅ (Timesheets merged in)
- **Removed the Timesheets tab** — its only real job was approve/reject, so that now lives in Attendance.
- **Approve / Reject on each record**: after an employee clocks out, admin reviews the **selfie + GPS location**, then Approves or Rejects. Status shows **Pending → Approved / Rejected**; still-on-the-clock rows show the live timer (no buttons until they clock out). A wrong decision can be flipped.
- **Reject voids the hours**: rejecting also marks the linked timesheet rejected, so **payroll skips those hours** — fake or unverified clock-ins don't get paid.
- **Employee notification**: approving/rejecting sends the worker an in-app notification ("Your attendance on … was approved/rejected").
- **Filters**: added **per-employee** and **per-company** dropdowns (admins excluded from the employee list), plus a **Clear** button. The date filter now **defaults to Today**, with **Today / All days** quick buttons.

## 4. Overtime Page — Filters & Tabs ✅
- Added the same **Employee + Company + Date** filters.
- Added **Pending / Reviewed toggle buttons** (with live counts) that **share the same filters**.
- The "Recently reviewed" table now shows the **Reason** and **Project** — so you can see *what* the overtime was for, not just the hours.

## 5. Payroll — Computed Salary View ✅
- Each pay run now **computes gross salary** = logged hours per band × the employee's hourly rate × the entity's OT/weekend/public-holiday multipliers.
- **"View salaries"** opens a per-employee breakdown (hours per band + gross pay + run total).
- Fixed a rate-date trap so a rate set today still applies to back-dated runs (no more silent $0).
- Tidied the Payroll **Actions** buttons (consistent sizing, right-aligned, Delete as a subtle icon).
