## 2. Mobile — Automatic clock-out after 8 hours ✅ (built)
If a worker forgets to clock out and a session runs past **8 hours**, the app now clocks them out automatically.
- **Capped at exactly 8 hours** (clock-out time = clock-in + 8h, not "now"), so a session left open for 15–20 hours still only logs an 8-hour shift — payroll never gets a bogus long day.
- No auto-overtime (the worker didn't confirm it).
- The employee is told via an in-app alert + an inbox notification.
- Fires on app open/refresh (main "forgot" case) and via the live on-screen timer if it crosses 8h while the app is open. Guards prevent double-closing.
- Files: `mobile/lib/queries.ts` (`autoClockOut`, `isSessionExpired`, shared `writeClockOut`), `mobile/app/(app)/index.tsx`.
